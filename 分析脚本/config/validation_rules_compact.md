# 验证规则（精简版）

> **用途**: 阶段4使用，用于报告验证
> **完整版**: [validation_rules.md](./validation_rules.md)

---

## YAML 格式验证清单（14项）

| # | 检查项 | 正确格式 |
|---|-------|---------|
| 1 | 字段数量正确 | 12个必需字段 + tags |
| 2 | 字段顺序正确 | 与模板序号1-14一致 |
| 3 | 无多余字段 | 仅包含定义的14个字段 |
| 4 | 字段名正确 | 使用 `评级分数`（非 `评分`） |
| 5 | 是否更新 | `是否更新: true`（布尔值） |
| 6 | 评级星级 | `评级星级: 4`（数字1-5） |
| 7 | 评级分数 | `评级分数: 3.65`（数字） |
| 8 | 报告链接 | `报告链接: "[[xxx-投资分析报告]]"` |
| 9 | 核心逻辑 | 字符串，≤50字 |
| 10 | 操作建议 | 字符串，≤50字 |
| 11 | 行业阶段 | 高成长赛道必填起步期/成长期/成熟期 |
| 12 | tags字段 | `tags: - 投资报告` |
| 13 | 文件保存成功 | 文件存在于正确路径 |
| 14 | 文件名格式正确 | `股票名称(代码)-投资分析报告.md` |

## Markdown 格式验证

| # | 检查项 | 正确格式 |
|---|-------|---------|
| 1 | 标题层级 | `#` → `## 一、` → `### 1.1` |
| 2 | 章节编号 | H2用中文数字，H3用数字 |
| 3 | 表格格式 | `| 列1 | 列2 |` + `|---|---|` |
| 4 | 代码块 | ` ``` ` 开头和结尾配对 |
| 5 | 列表格式 | `- ` 或 `1. ` 后有空格 |

## 章节结构验证

必需章节：公司概况、财务分析、行业分析、竞争分析、估值分析、投资建议-综合评级、投资逻辑、风险提示、投资论点总结、免责声明

---

## 数据完整性验证

- 核心财报数据覆盖完整
- 行业数据覆盖完整
- 估值数据覆盖完整
- 加分/扣分依据明确
- 股票类型识别依据明确
