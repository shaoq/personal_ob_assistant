## 🎯 阶段目标

基于评分结果生成报告大纲，准备各章节的关键数据点，输出 `phase3_1_state.json` 中间状态文件。

**Context 消耗预估**: ~8k-10k tokens

---

## 📥 输入

- 阶段2摘要文件： `$INVESTMENT_HANDBOOK/.analysis_states/{股票代码}/phase2_summary.json`
- 阶段2状态文件： `$INVESTMENT_HANDBOOK/.analysis_states/{股票代码}/phase2_state.json`

---

## ⚠️ 关键约束速查表

| 约束项 | 要求 | 违反后果 |
|--------|------|---------|
| **数据完整性** | 必须读取完整的阶段2状态 | 报告框架缺失 |
| **章节规划** | 必须规划所有必需章节 | 报告内容缺失 |

---

## 📋 步骤清单

- [ ] 步骤1：读取 phase2_summary.json（精简摘要）
- [ ] 步骤2：按需加载 phase2_state.json（详细数据）
- [ ] 步骤3：规划报告章节结构
- [ ] 步骤4：准备各章节关键数据点
- [ ] 步骤5：生成 phase3_1_state.json

## 步骤 1：读取 phase2_summary.json

**操作内容**：

1. 读取阶段2摘要文件（约500 tokens）：
   ```
   $INVESTMENT_HANDBOOK/.analysis_states/{股票代码}/phase2_summary.json
   ```

2. 提取关键信息：
   - 股票基本信息（代码、名称）
   - 股票类型
   - 五维度评分摘要
   - 综合评分与评级
   - 成长系数

3. 验证状态文件完整性：
   - status = "completed"

**输出结果**：
- ✅ phase2_summary.json 读取成功
- ✅ 关键信息提取完成

---

## 步骤 2：按需加载 phase2_state.json

**操作内容**：

仅当需要详细数据时，读取完整的 phase2_state.json（按需加载）：

| 需要加载的情况 | 加载内容 |
|---------|---------|
| 评分详情不清晰 | 加载 five_dimension_scores 部分 |
| 加扣分依据不明确 | 加载 conditional_data 部分 |
| 权重体系需要确认 | 加载 comprehensive_scoring 部分 |

**输出结果**：
- ✅ 按需加载详细数据完成

---

## 步骤 3：规划报告章节结构

**必需章节规划**：

| 序号 | 章节名称 | 关键内容 | 数据来源 |
|------|---------|---------|---------|
| 一 | 公司概况 | 基本信息、主营业务 | phase1_state |
| 二 | 财务分析 | 核心财务数据表格 | phase1_state |
| 三 | 行业分析 | 行业规模、竞争格局 | phase1_state |
| 四 | 竞争分析 | 竞争对手、市场份额 | phase1_state |
| 五 | 估值分析 | PE/PB/DCF等 | phase1_state |
| 六 | 投资建议-综合评级 | 五维度评分表、综合评分计算 | phase2_state |
| 七 | 投资逻辑 | 3-5点核心逻辑 | phase2_state |
| 八 | 风险提示 | 主要风险因素 | phase1_state |
| 九 | 投资论点总结 | 看多/看空论点 | phase2_state |
| + | 免责声明 | 标准免责声明 | 模板 |

**输出结果**：
- ✅ 章节结构规划完成
- ✅ 每章节关键数据点已标记

---

## 步骤 4：准备各章节关键数据点

**4.1 公司概况数据点**

| 数据项 | 来源 |
|--------|------|
| 股票代码/名称 | phase2_summary |
| 所属行业 | phase1_state |
| 主营业务描述 | phase1_state |
| 总市值 | phase1_state |

**4.2 财务分析数据点**

| 数据项 | 来源 |
|--------|------|
| 营业收入（近3年） | phase1_state |
| 净利润（近3年） | phase1_state |
| 毛利率/净利率 | phase1_state |
| 资产负债率 | phase1_state |
| ROE | phase1_state |
| 现金流 | phase1_state |

**4.3 行业分析数据点**

| 数据项 | 来源 |
|--------|------|
| 行业CAGR | phase1_state |
| 政策支持情况 | phase1_state |
| 竞争格局 | phase1_state |
| 行业阶段（如适用） | phase1_state |

**4.4 投资建议数据点**

| 数据项 | 来源 |
|--------|------|
| 五维度评分详情 | phase2_state |
| 权重体系 | phase2_summary |
| 成长系数（如适用） | phase2_state |
| 综合评级 | phase2_summary |

**4.5 核心逻辑和操作建议**

**核心逻辑**（一句话总结，不超过50字）：
- 从投资逻辑章节提炼核心投资价值
- 格式：`"核心优势 + 催化剂 + 预期"`

**操作建议**（一句话建议，不超过50字）：
- 基于评级和风险提示给出操作建议
- 格式：`"时间维度 + 关键关注点 + 风险提示"`

**输出结果**：
- ✅ 各章节关键数据点已准备
- ✅ 核心逻辑已提炼
- ✅ 操作建议已准备

## 步骤 5：生成 phase3_1_state.json

**操作内容**：

1. 生成 `phase3_1_state.json` 文件，包含：
   - 股票基本信息
   - 股票类型
   - 五维度评分摘要
   - 综合评级
   - 报告章节规划
   - 各章节关键数据点索引
   - **核心逻辑**（一句话总结，不超过50字）
   - **操作建议**（一句话建议，不超过50字）

2. 设置状态：
   ```json
   {
     "status": "completed",
     "next_phase": "phase3_2"
   }
   ```

3. 保存到：
   ```
   $INVESTMENT_HANDBOOK/.analysis_states/{股票代码}/phase3_1_state.json
   ```

**输出结果**：
- ✅ phase3_1_state.json 已生成
- ✅ 状态文件保存成功
