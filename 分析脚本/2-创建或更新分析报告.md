# 股票投资分析报告生成流程

> **文档版本**: 2026-03-05
> **最后更新**: 强化评分逻辑 - 五维度评分必填，综合评分必须由五维度计算得出，禁止仅根据评级推算评分

---

### 步骤 1：确定 Obsidian Vault 路径

在开始分析前，首先确定用户的 Obsidian Vault 实际路径：

**自动检测方式（按优先级尝试）：**

1. 检查常见路径位置：
   - macOS: `~/Documents/Obsidian Vault/` 或 `~/Library/Mobile Documents/iCloud~md~obsidian/Documents/`
   - Windows: `C:\Users\<用户名>\Documents\Obsidian Vault\`
   - Linux: `~/Documents/Obsidian Vault/`

2. 如果自动检测失败，主动询问用户：
   - "请提供您的 Obsidian Vault 的完整路径"

3. 确认路径有效性后，后续操作均使用此路径变量，路径变量标记为：`$OBSIDIAN_VAULT`

### 步骤 2：等待用户输入

等待用户提供待分析的股票列表，格式为：`股票代码(股票名称)`

示例输入：
- 单只股票：`301292(海科新源)`
- 多只股票：`301292(海科新源)，600519(贵州茅台)，000858(五粮液)`

### 步骤 3：前置字段验证（关键步骤）

> **重要**: 此步骤为新增验证机制，确保生成的报告字段与 base 文件公式一致。

**3.1 读取 base 文件**

在生成任何报告前，必须先读取：
`$OBSIDIAN_VAULT/投资手册/投资分析报告汇总.base`

**3.2 提取字段要求**

从 base 文件中提取：
1. **formulas** 中引用的字段名 → 这些是报告 YAML 必须包含的字段
2. **properties** 中定义的字段 → 确认字段命名规范
3. **数据类型要求** → 根据公式逻辑判断字段类型

**3.3 字段验证清单**

| 检查项 | 验证内容 | 通过标准 |
|--------|----------|----------|
| 字段名正确性 | YAML 字段名与 base 公式引用完全一致 | 无拼写差异 |
| 数据类型正确 | 布尔字段用 `true/false`，数值字段用数字 | 非字符串混淆 |
| 必需字段完整 | 所有公式依赖的字段都存在 | 无缺失字段 |

### 步骤 4：逐只股票执行分析

对每只股票依次执行以下操作，**务必逐只执行，不要并行**：

**4.1 调用分析 Skill**

执行命令：`equity-research:thesis <股票代码>(<股票名称>)`

示例：`equity-research:thesis 301292(海科新源)`

数据要求：必须全面查询所有可用数据，必须补全所有缺失数据字段，必须进行综合分析。

**4.2 报告内容要求**

除了 skill 自带内容外，必须包含但不限于以下章节：

- **投资建议-综合评级**：必须包含以下评分内容：
  - **五个维度评分**（每项 1-5 分）：
    - 行业前景：X 分
    - 公司竞争力：X 分
    - 成长性：X 分
    - 财务健康度：X 分
    - 估值合理性：X 分
  - **综合评分**：五维度评分的平均值（保留 1 位小数）
  - **综合评级**：根据评分范围对照"投资评级定义"文件给出对应评级及评级理由
- **投资逻辑**：核心投资逻辑的 3-5 点总结
- **投资论点总结**：看多/看空的关键论点归纳

以满足 base 文件的要求。

**4.3 文件保存规则**

保存路径：`$OBSIDIAN_VAULT/投资手册/投资报告/<股票名称>(<股票代码>)-投资分析报告.md`

示例：`$OBSIDIAN_VAULT/投资手册/投资报告/海科新源(301292)-投资分析报告.md`

文件处理规则：
- 文件不存在时创建新文件
- 文件已存在时全量更新内容

### 步骤 5：YAML 结构适配（核心）

> **⚠️ 关键原则**：以 base 文件公式为"真相源"，所有字段定义必须以它为准。

**5.1 必需字段清单（必须全部包含）**

| 字段名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| 股票代码 | string | 6位数字代码 | "300353" |
| 股票名称 | string | 股票中文名称 | "东土科技" |
| 综合评级 | string | 评级定义文件中的评级 | "增持" |
| **评分** | **number** | **五维度总评分（1-5分）** | **4** |
| 所属行业 | string | 主营业务所属行业 | "工业互联网" |
| 市值 | string | 总市值数值（不带"亿"） | "153" |
| 报告日期 | string | 分析报告日期 | "2026-03-04" |
| 核心逻辑 | string | 核心投资逻辑一句话总结 | "工业互联网底层根技术龙头..." |
| 操作建议 | string | 操作建议一句话 | "中长线布局，关注业绩兑现" |
| **是否更新** | **boolean** | **新建或更新时设为 `true`** | **true** |
| **报告链接** | **string** | **Obsidian 内部链接格式** | **"[[东土科技(300353)-投资分析报告]]"** |

**5.2 字段对应 base 公式说明**

```yaml
# base 文件中的公式逻辑：

# 评分数值：优先使用"评分"字段，若无则根据"综合评级"推算
formula.评分数值: if(评分, 评分, if(综合评级 == "强力买入", 5, ...))

# 评级图标：根据"综合评级"文字匹配图标
formula.评级图标: if(综合评级 == "强力买入", "🚀", ...)

# 更新图标：根据"是否更新"布尔值显示图标
formula.更新图标: if(是否更新, "✅", "⚪")

# 市值格式化：在"市值"数值后添加"亿"
formula.市值亿元: if(市值, 市值 + " 亿", "")

# 报告链接：直接显示YAML中的链接字段
报告链接 -> 生成可点击的跳转链接
```

**5.3 评分与综合评级的关系**

> **关键原则**：`评分` 字段**必须**基于五维度评分计算得出，**不可省略**，**不可仅根据综合评级推算**。

**评分计算方式（强制执行）**：
1. 首先对五个维度分别打分（每项 1-5 分）：
   - 行业前景
   - 公司竞争力
   - 成长性
   - 财务健康度
   - 估值合理性
2. **综合评分** = 五维度评分的平均值（保留 1 位小数）
3. 示例：(5 + 5 + 4 + 3 + 3) / 5 = 4.0

**综合评级对照表**：

| 综合评分范围 | 综合评级 |
|-------------|----------|
| 4.5 - 5.0 | 强力买入 |
| 3.5 - 4.4 | 买入 |
| 2.5 - 3.4 | 增持 |
| 1.5 - 2.4 | 观望 |
| 0 - 1.4 | 回避 |

> **注意**：综合评级由综合评分决定，而非相反。

**评级图标对照表**：

| 综合评级 | 评级图标 |
|----------|----------|
| 强力买入 | 🚀 |
| 买入 | ⭐⭐⭐⭐⭐ |
| 增持 | ⭐⭐⭐⭐ |
| 观望 | ⭐⭐⭐ |
| 回避 | ⚠️ |

**5.5 YAML 字段填写顺序**

为确保评分准确性，YAML 字段按以下逻辑顺序填写：

1. **先填写五维度评分**（在报告正文中体现）
2. **计算综合评分** = 五维度平均值
3. **根据综合评分确定综合评级**
4. **填写 YAML 字段**：`评分` 和 `综合评级`

### 步骤 6：生成后验证

**6.1 自动验证检查项**

| 检查项 | 验证方法 | 预期结果 |
|--------|----------|----------|
| 更新图标显示 | 检查 base 视图 | 显示 ✅ 而非 ⚪ |
| 报告链接可点击 | 点击链接 | 跳转到对应报告文件 |
| **五维度评分完整** | **检查报告正文** | **五个维度均有评分** |
| **综合评分计算正确** | **验证五维度平均值** | **YAML 评分与计算值一致** |
| **评分与评级匹配** | **对照评分范围表** | **评级与评分范围对应** |
| 评级图标显示 | 检查 base 视图 | 显示对应星级图标 |

**6.2 如验证失败**

1. 检查 YAML 字段名是否与 base 公式引用完全一致
2. 确认 `是否更新` 是布尔值 `true`，而非字符串 `"是"`
3. **确认 `评分` 是数字类型（如 `4.0`），而非字符串（如 `"4"` 或 `"4.0分"`）**
4. **确认五维度评分均在报告正文中清晰列出**
5. **确认 `评分` 值为五维度评分的平均值，而非随意填写**
6. 确认 `报告链接` 使用 Obsidian 双中括号链接格式

## 常见问题与解决方案

### 问题 1：更新图标显示为 ⚪ 而非 ✅

**原因**：`是否更新` 字段值为字符串 `"是"` 而非布尔值 `true`

**解决**：将 YAML 中的 `是否更新: "是"` 改为 `是否更新: true`

### 问题 2：报告链接列显示为空

**原因**：缺少 `报告链接` 字段

**解决**：添加 `报告链接: "[[股票名称(股票代码)-投资分析报告]]"` 字段

### 问题 3：缺少五维度评分或综合评分

**原因**：报告未按要求包含完整的五维度评分

**解决**：确保报告"投资建议-综合评级"章节包含：
- 行业前景评分（1-5分）
- 公司竞争力评分（1-5分）
- 成长性评分（1-5分）
- 财务健康度评分（1-5分）
- 估值合理性评分（1-5分）
- 综合评分（五维度平均值）

### 问题 4：评分与评级不匹配

**原因**：`综合评级` 不是根据 `评分` 范围确定

**解决**：
1. 计算五维度评分平均值得到综合评分
2. 根据综合评分范围对照表确定综合评级
3. 综合评级由评分决定，不可跳过评分直接定级
