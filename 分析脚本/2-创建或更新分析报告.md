## ⚠️ 关键约束速查表

> **重要**：以下约束在执行过程中必须严格遵守，违反将导致错误结果

| 约束项 | 要求 | 违反后果 | 检查时机 |
|--------|------|---------|---------|
| **P2.5数据完整性** | 类型识别必需数据不可缺失>30% | 类型识别失败 | 步骤5执行前 |
| **输入格式** | **每次只输入一只股票**，处理完成后再输入下一只 | 上下文混乱，状态丢失 | 步骤2执行时 |
| **循环返回点** | 清理上下文后必须返回步骤3（不是步骤2） | 配置文件丢失，分析失败 | 步骤12用户确认后 |
| **数据完整性** | P1-P3+P2.5必需数据不可缺失>30% | 评分不准确，验证失败 | 步骤5执行前 |
| **YAML格式** | 严格遵循字段类型定义 | base视图显示异常 | 步骤9验证时 |
| **上下文清理** | 每只股票后执行 `/clear` | 上下文溢出，性能下降 | 步骤11完成后 |
| **专用Skill** | 必须使用 `equity-research:thesis` | 报告格式不符合要求 | 步骤7执行时 |
| **文件保存** | 必须保存到 `$INVESTMENT_HANDBOOK/投资报告/` | 报告丢失，无法汇总 | 步骤8完成后 |
| **字段名一致性** | 使用 `是否更新`（非 `是否有更新`） | Base视图更新图标显示异常 | 步骤8 YAML填写时 |
| **后续动作** | 完成后等待用户输入，不自动执行其他操作 | 执行非预期任务 | 步骤12结束后 |

## 🔄 流程总览

```
【首次进入】
步骤1：确定路径（执行一次）
    ↓
步骤2：等待用户输入股票
    ↓
【每只股票循环】
步骤3：读取核心配置文件 ←┐
    ↓                    │
步骤4-6：数据准备        │
    ↓                    │
步骤7-8：报告生成        │（清理上下文后
    ↓                    │ 需重新执行）
步骤9-10：验证修复       │
    ↓                    │
步骤11：清理上下文       │
    ↓                    │
步骤12：完成询问 ────────┘
    ↓
用户确认继续 → 等待新股票输入 → 返回步骤3
```

> **关键说明**：
> - **步骤1**：仅在首次进入时执行一次
> - **步骤2**：等待用户输入，不涉及配置读取
> - **步骤3**：每次分析新股票时都必须执行（因为步骤11清理了上下文）
> - **步骤4-12**：针对当前股票执行

## 步骤 1：确定 Obsidian Vault 路径

**执行时机**：流程开始时执行一次

**操作内容**：

在开始分析前，首先确定用户的 Obsidian Vault 实际路径：

**自动检测方式（按优先级尝试）：**

1. 检查常见路径位置：
   - macOS: `~/Documents/Obsidian Vault/` 或 `~/Library/Mobile Documents/iCloud~md~obsidian/Documents/`
   - Windows: `C:\Users\<用户名>\Documents\Obsidian Vault\`
   - Linux: `~/Documents/Obsidian Vault/`

2. 如果自动检测失败，主动询问用户：
   - "请提供您的 Obsidian Vault 的完整路径"

3. 确认路径有效性后，后续操作均使用此路径变量，路径变量标记为：`$OBSIDIAN_VAULT`

**路径选择优先级**：

| 优先级 | 路径 | 说明 |
| :--- | :--- | :--- |
| 1（最高） | `$OBSIDIAN_VAULT/personal_ob_assistant/投资手册/` | 如果此路径存在，优先使用 |
| 2（备选） | `$OBSIDIAN_VAULT/投资手册/` | 仅当 personal_ob_assistant 目录不存在时使用 |

**目标路径变量**：后续步骤中的路径将使用 `$INVESTMENT_HANDBOOK` 变量替代。

**输出结果**：
- ✅ 确定 `$OBSIDIAN_VAULT` 变量
- ✅ 确定 `$INVESTMENT_HANDBOOK` 变量（根据路径选择优先级）
- ✅ 路径有效性验证通过

## 步骤 2：等待用户输入

**执行时机**：每次分析新股票时

**操作内容**：

等待用户提供待分析的股票，格式为：`股票代码(股票名称)`

**示例输入**：
- 单只股票：`301292(海科新源)`

**处理规则**：
- **每次只处理一只股票**
- 处理完成并验证通过后，向用户确认是否继续分析下一只股票
- 用户确认继续并输入新股票后，**返回步骤3**重新读取配置文件（因为步骤11清理了上下文）

> ⚠️ **重要**：不支持一次性输入多只股票（如 `301292(海科新源)，600519(贵州茅台)`）。
> 这种输入方式会导致状态丢失，无法依次处理。请每次只输入一只股票。

**输出结果**：
- ✅ 获得单只股票信息
- ✅ 输入格式验证通过（单只股票）

**下一步操作**：
- 📍 立即执行 [步骤3：读取核心配置文件](#步骤-3-读取核心配置文件)

**操作内容**：

**3.2 读取行业分类配置文件（新增）**

读取：`分析脚本/行业分类配置.md`

**提取内容**：
1. **行业阶段定义** → 用于判断行业所处阶段（起步期/成长期/成熟期）
2. **高成长赛道清单** → 用于判断公司所属行业是否为高成长赛道

**3.3 读取 base 文件**

2. **properties** 中定义的字段 → 确认字段命名规范
3. **数据类型要求** → 根据公式逻辑判断字段类型

**3.4 字段验证清单**

| 检查项 | 验证内容 | 通过标准 |
|--------|----------|----------|

| 数据类型正确 | 布尔字段用 `true/false`，数值字段用数字 | 非字符串混淆 |
| 必需字段完整 | 所有公式依赖的字段都存在 | 无缺失字段 |

**输出结果**：
- ✅ 评级定义文件内容已加载
- ✅ 行业分类配置已加载（新增）

### P2.1 - 行业属性判断 + 行业阶段判断（三层识别第一层）

> **执行时机**：获取"所属行业"和"主营业务描述"后立即执行

**前置条件**：在执行行业属性判断前，确保已获取：
- 所属行业（P2）
- 主营业务描述（P2）
- 高成长赛道清单（从行业分类配置文件提取）
- 行业阶段定义（从行业分类配置文件提取）

**判断流程**：

```
【行业属性判断】
1. 精确匹配：
   IF 所属行业 ∈ 高成长赛道清单
       → 行业属性 = "高成长赛道" ✅
       → 记录匹配的行业名称

2. 关键字匹配：
   ELSE IF 主营业务描述 包含 高成长赛道识别关键字
       → 行业属性 = "高成长赛道" ✅
       → 记录匹配的关键字

3. 默认：
   ELSE
       → 行业属性 = "普通行业"

【行业阶段判断】（仅对高成长赛道）
IF 行业属性 = "高成长赛道"
   IF 渗透率<5% AND (技术处于验证期 OR 市场规模<1000亿)
       → 行业阶段 = "起步期"
   ELSE IF 渗透率5%-20% OR (技术规模化应用 AND 政策体系完善)
       → 行业阶段 = "成长期"
   ELSE
       → 行业阶段 = "成熟期"
ELSE
   → 行业阶段 = null（普通行业不需要判断阶段）
```

**行业阶段判断所需数据**：
| 数据项 | 来源 | 用途 |
| :--- | :--- | :--- |
| 行业渗透率 | P2.5 | 判断行业阶段 |
| 技术成熟度 | P2.5 | 判断行业阶段 |
| 市场规模 | P2.5 | 判断行业阶段 |
| 政策支持度 | P2 | 判断行业阶段 |

**高成长赛道识别关键字**（从行业分类配置文件提取）：
- AI/智能化：人工智能、AI、大模型、机器人、脑机
- 算力基础设施：数据中心、智算中心、服务器、液冷
- 硬科技：半导体、芯片、集成电路、量子、光刻
- 新能源：锂电、光伏、风电、储能
- 高端制造：工业机器人、数控机床、伺服
- 生物医药：创新药、生物药、CXO、CRO
- 未来产业：商业航天、低空经济、无人机

**输出结果**：
- ✅ 行业属性已确定（高成长赛道/普通行业）
- ✅ 行业阶段已确定（起步期/成长期/成熟期/null）
- ✅ 匹配依据已记录

## 步骤 5：股票类型识别（三层复合决策树）

**执行时机**：步骤4完成后，基于行业属性和财务数据判断

**操作内容**：

> **重要**：股票类型识别采用**三层复合体系**，必须严格按决策树执行。

### 前置检查：数据就绪确认

在执行类型识别前，确认以下数据已获取：

| 检查项 | 数据来源 | 状态 |
| :--- | :--- | :--- |
| 行业属性已确定 | P2/P2.1 | ☐ |
| 净利润近3年数据 | P1/P2.5 | ☐ |
| 营收CAGR | P1 | ☐ |
| 利润增速标准差 | P2.5 | ☐ |
| 新业务相关数据（如适用） | P2.5 | ☐ |
| 前瞻指标数据（如适用） | P2.5 | ☐ |

> **重要**：若数据不完整，返回步骤4补充获取P2.5数据后再执行类型识别

### 三层识别流程

> **重要**：股票类型识别采用**三层复合体系**，必须严格按决策树执行。
> **注意**：已新增「行业阶段判断」作为第一层的一部分
```
第一层：行业属性判断 + 行业阶段判断（步骤4.1已完成）
    ↓
第二层：财务指标验证（本步骤执行）
    ↓
第三层：前瞻指标验证（仅对潜力成长股）
```

### 完整决策树

> **重要**：必须严格按优先级顺序判断
> **重要**：已新增「早期成长股」和「早期转型股」类型
```
【第一层：行业属性 + 行业阶段】（已在步骤4.1确定）
行业属性 = "高成长赛道" 或 "普通行业"
行业阶段 = "起步期" 或 "成长期" 或 "成熟期" 或 null（普通行业不需要判断阶段）

【第二层：财务指标验证】

【优先级1：亏损股】
IF 净利润（近3年平均）< 0
    → 最终类型 = 亏损股

【优先级2：周期股】
ELSE IF 近3年利润增速标准差 > 行业平均 × 2
    → 最终类型 = 周期股

【优先级3：转型股/早期转型股】
ELSE IF 满足转型股条件（见上表）
    IF 新业务所处行业阶段 = "起步期"
        → 最终类型 = 早期转型股
    ELSE
        → 最终类型 = 转型股

【优先级3.5：潜力转型股】（新增）
ELSE IF 满足以下2条中任意1条：
    - 新业务营收占比 3%-10% 且 新业务增速 > 100%
    - 新业务营收占比 > 3% 且 新业务所在行业CAGR > 15%
    → 最终类型 = 潜力转型股

【优先级4：早期成长股】（新增 - 重要）
ELSE IF 行业属性 = "高成长赛道" AND 行业阶段 = "起步期"
    IF 营收CAGR > 10% AND 公司地位领先（份额前5 或 头部客户验证）
        → 最终类型 = 早期成长股
    ELSE IF 营收CAGR > 0% AND 潜力指标支撑
        → 最终类型 = 潜力成长股
    ELSE
        → 最终类型 = 成熟股

> **注意**：起步期行业的判断依据
> - 渗透率<5% OR 技术处于验证期 OR 市场规模<1000亿
> - 公司地位领先：市场份额前5 或 已获得头部客户验证 或 拥有核心专利/独家技术
> - 潜力指标：在手订单、产能储备、技术储备等（参考投资评级定义文件）

【优先级5-7：成长类型判断（分行业属性）】

ELSE IF 行业属性 = "高成长赛道"
    IF 营收CAGR > 50%
        → 最终类型 = 高成长股
    ELSE IF 营收CAGR > 20%
        → 最终类型 = 成长股
    ELSE IF 营收CAGR > 0%
        → 候选"潜力成长股"（进入第三层）
    ELSE
        → 最终类型 = 成熟股

ELSE（普通行业）
    IF 营收CAGR > 50%
        → 最终类型 = 高成长股
    ELSE IF 营收CAGR > 15%
        → 最终类型 = 成长股
    ELSE
        → 最终类型 = 成熟股

【第三层：前瞻指标验证】（仅对候选"潜力成长股"）
IF 满足任一前瞻指标：
    - 在手订单/年收入 > 100%
    - 产能扩张计划 > 50%
    - 行业渗透率 < 20% 且公司为头部
    - 客户结构优化（新增头部客户）
    - 业务量增速 > 30%（机柜数/用户数/出货量等）
    → 最终类型 = 潜力成长股
ELSE
    → 最终类型 = 成熟股
```

> **注意**：潜力成长股和早期成长股的区别
> - **潜力成长股**：行业处于成长期（渗透率5%-20%），但公司增速放缓（0%-20%），需要潜力指标支撑
> - **早期成长股**：行业处于起步期（渗透率<5%），公司增速>10%且有领先地位，增速受行业基数限制

### 股票类型汇总（10种类型）

| 股票类型 | 识别条件 | 特征描述 | 适用权重体系 |
| :--- | :--- | :--- | :--- |
| **早期成长股** | 高成长赛道起步期 + 增速>10% + 鸒头验证/份额前5 | 行业早期，增长受基数限制 | 早期成长股权重 |
| **早期转型股** | 转向起步期行业 + 新业务占比>3% + 增速>30% | 转型方向处于早期 | 早期转型股权重 |
| **高成长股** | 营收增速>50% | 业绩爆发期 | 高成长股权重 |
| **成长股** | 营收增速20%-50%（高成长赛道）或 15%-50%（普通行业） | 成长动能强劲 | 成长股权重 |
| **潜力成长股** | 高成长赛道 + 营收增速0%-20% + 潜力指标支撑 | 产能爬坡期 | 潜力成长股权重 |
| **成熟股** | 增速<20%（高成长赛道）或 <15%（普通行业） | 稳定经营 | 成熟股权重 |
| **周期股** | 利润增速标准差 > 行业平均 × 2 | 盈利随周期波动 | 周期股权重 |
| **亏损股** | 净利润（近3年平均）< 0 | 投入期未盈利 | 亏损股权重 |
| **转型股** | 满足转型股条件（见上） | 向高成长赛道转型 | 转型股权重 |
| **潜力转型股** | 新业务占比3%-10%且增速>100%，或新业务占比>3%且所在行业cAGR>15% | 早期转型， 新业务高增长 | 潜力转型股权重 |

```

4. 最终评分计算

   示例：3.65 × 1.10 = 4.02分

5. 延伸业务加分（如适用）

## 步骤 10：异常处理（仅在验证失败时执行）

**10.2 股票类型识别冲突**

**场景**：一只股票同时满足多个类型的条件

**处理方式**：
```
识别优先级（从高到低）：
1. 亏损股（净利润<0）
2. 周期股（利润波动大）
3. 转型股（新业务占比高）
4. 高成长股（增速>50%）
5. 成长股（增速>15%）
6. 成熟股（其他）

```

**10.3 评分计算错误**

**常见错误**：
- 评级星级与评级分数不匹配

**处理方式**：
1. 重新识别股票类型（步骤5）
2. 重新选择对应权重体系
3. 重新计算综合评分
4. 重新确定评级星级

6. 重新执行步骤9验证

1. 检查路径是否正确

## 🆘 常见问题与解决方案

### 问题 1：更新图标显示为 ⚪ 而非 ✅

**原因**：`是否更新` 字段值为字符串 `"是"` 而非布尔值 `true`

**解决**：将 YAML 中的 `是否更新: "是"` 改为 `是否更新: true`

**预防**：使用步骤8提供的标准化 YAML 模板

---

### 问题 2：报告链接列显示为空

**原因**：缺少 `报告链接` 字段

**解决**：添加 `报告链接: "[[股票名称(股票代码)-投资分析报告]]"` 字段

**预防**：使用步骤8提供的标准化 YAML 模板

---

### 问题 3：缺少五维度评分或评级分数

**原因**：报告未按要求包含完整的五维度评分

**解决**：确保报告"投资建议-综合评级"章节包含：
- 行业前景评分（1-5分）
- 公司竞争力评分（1-5分）
- 成长性评分（1-5分）
- 财务健康度评分（1-5分）
- 估值合理性评分（1-5分）
- 评级分数（五维度平均值）

**预防**：使用步骤7.2的评分表格模板

---

### 问题 4：评级星级与评级分数不匹配

**原因**：
- `评级星级` 不是根据 `评级分数` 范围确定
- 未使用正确的权重体系
- 股票类型识别错误

**解决**：
1. 重新识别股票类型（参考步骤5决策树）
2. 使用对应权重计算加权综合评分（参考评级定义文件）
3. 根据步骤8.2的评分范围对照表确定评级星级
4. 评级由综合评分决定，必须严格遵循评级定义文件

**预防**：严格按照步骤5-8的顺序执行，不要跳过股票类型识别步骤

---

### 问题 5：评级图标显示"未评级"

**原因**：缺少 `评级星级` 字段或值不在 1-5 范围内

**解决**：
1. 确保 YAML 中包含 `评级星级` 字段
2. 确认 `评级星级` 值为 1-5 的整数

**预防**：使用步骤8提供的标准化 YAML 模板

---

### 问题 6：必需数据缺失过多

**原因**：P1-P3必需数据缺失超过30%

**解决**：
1. 中止当前分析
2. 向用户报告缺失的数据项
3. 等待用户补充数据或确认继续

**预防**：在步骤4严格执行数据查询，遇到缺失立即标注

### 问题 7：输入多只股票导致处理中断

**原因**：一次性输入多只股票（如 `301292(海科新源)，600519(贵州茅台)`），导致执行 `/clear` 后状态丢失

**解决**：
1. 确保每次只输入一只股票
2. 处理完成后等待系统询问是否继续
3. 在确认继续后，再输入下一只股票

**预防**：严格遵守步骤2的输入格式要求，每次只输入一只股票

---

### 问题 8：字段名使用错误（如"是否有更新"而非"是否更新"）

**原因**：未仔细阅读 base 文件的字段定义
**解决**：
1. 重新读取 base 文件（步骤3.2）
2. 确认字段名与 base 公式引用完全一致
3. 修正 YAML 字段名
**预防**：使用步骤8提供的标准化 YAML 模板，或使用步骤0的字段名对照表

### 问题 9：类型识别时数据缺失

**原因**：P2.5类型识别必需数据未完整获取

**表现**：
- 无法判断是否为周期股（缺利润增速标准差）
- 无法判断是否为转型股（缺新业务相关数据）
- 无法判断是否为潜力成长股（缺前瞻指标）

**解决**：
1. 检查步骤4是否完整获取P2.5数据
2. 补充缺失数据项
3. 重新执行步骤5类型识别

**预防**：在步骤4严格执行P2.5数据清单检查，确保类型识别所需数据完整获取

| **潜力转型股** | **新业务占比3%-10%且增速>100%，或新业务占比>3%且行业CAGR>15%** | **3.5** | **早期转型，新业务高增长** |
| **早期成长股** | **高成长赛道起步期 + 增速>10% + 鸒头验证/份额前5** | **4** | **行业早期，增长受基数限制** |
| 高成长股 | 营收CAGR > 50% | 4 | 业绩爆发期 |
| 成长股 | 营收CAGR 20%-50%（高成长赛道）或 15%-50%（普通行业） | 5 | 成长动能强劲 |
| 潜力成长股 | 高成长赛道 + 营收CAGR 0%-20% + 前瞻指标支撑 | 6 | 产能爬坡期 |
| 成熟股 | 其他 | 7（最低） | 稳定经营 |

**三层识别要点**：
1. **第一层**：行业属性 + 行业阶段（起步期/成长期/成熟期）
2. **第二层**：财务指标验证（增速、利润稳定性、转型数据）
3. **第三层**：前瞻指标微调（仅对潜力成长股）
> **注意**：早期成长股不需要第三层验证

### 评级等级（5级）

| 评级 | 综合评分范围 | 评级星级 | 预期收益特征 |

| 买入 | 4.0 - 4.4 | 5 | 涨幅高于基准 10%-20% |
| 增持 | 3.5 - 3.9 | 4 | 小幅跑赢或优于大盘 5%-15% |
