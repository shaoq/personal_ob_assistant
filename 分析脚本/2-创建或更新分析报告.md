### 步骤 4：逐只股票执行分析

对每只股票依次执行以下操作，**务必逐只执行，不要并行**：

**4.1 调用分析 Skill**

执行命令：`equity-research:thesis <股票代码>(<股票名称>)`

示例：`equity-research:thesis 301292(海科新源)`

**数据查询要求**（必须严格遵循）：

1. **按优先级顺序查询**：严格按照步骤3.6数据查询清单的优先级顺序（P1→P2→P3→P4→P5/P6）
2. **必需数据优先**：必须首先完成P1-P3必需数据查询，缺一不可
3. **条件必需数据**：根据初步判断的股票类型，查询对应的条件必需数据：
   - 若疑似转型股 → 必须查询P5转型股专属数据
   - 若疑似周期股 → 必须查询P6周期股专属数据
4. **加分/扣分数据**：尽可能收集P4加分/扣分因素数据，确保评分有充分依据
5. **数据缺失处理**：
   - 必需数据缺失时，在报告中明确标注"数据缺失"
   - 评分时需说明数据缺失对判断的影响
   - 无法评分的维度应说明原因

> **注意**：数据完整性直接影响评分准确性，宁可多查询也不要遗漏关键数据。

**4.2 报告内容要求**

除了 skill 自带内容外，必须包含但不限于以下章节：

- **股票类型识别**：明确说明股票属于哪种类型（共6种，详见 [[投资评级定义#股票类型识别]]），并给出识别依据
- **投资建议-综合评级**：必须包含以下评分内容：
  - **五个维度评分**（每项 1-5 分），需包含：
    - 基础评分依据（引用评级定义文件中的量化标准）
    - 加分/扣分因素说明
    - 计算过程（展示完整计算公式）
    - 最终评分（确保在1-5分范围内）
  - **综合评分**：
    - 说明使用的权重体系（根据股票类型）
    - 加权计算过程
    - 成长系数判断（如适用）：
      - 高成长股（增速>50%）或转型股（新业务增速>50%）需说明成长系数
      - 其他股票类型成长系数为1.00（不调整）
    - 最终评分计算：基础综合评分 × 成长系数（上限5.0分）
  - **综合评级**：根据评分范围对照评级定义给出对应评级及评级理由
    - 若应用了成长系数（≠1.00），需说明成长系数的确定依据
- **投资逻辑**：核心投资逻辑的 3-5 点总结
- **投资论点总结**：看多/看空的关键论点归纳

以满足 base 文件的要求。

**4.2.1 表格样式规范**

为确保表格在 Obsidian 中正确渲染，报告中的所有表格必须遵循以下规范：

1. **表格对齐格式**：分隔行使用对齐的冒号格式，例如：
   ```markdown
   | 列名 | 列名 | 列名 |
   | :--- | :--- | :--- |
   | 内容 | 内容 | 内容 |
   ```

2. **列宽控制**：
   - 单列内容不超过 30 个中文字符（约 60 字节）
   - 超长内容应换行或使用列表形式呈现
   - 数值列右对齐（使用 `---:` 格式）

3. **表格类型与适用场景**：

   | 表格类型 | 适用场景 | 示例 |
   | :--- | :--- | :--- |
   | 简单对照表 | 2-4 列的键值对 | 基本信息、财务指标 |
   | 评分表格 | 五维度评分等固定格式 | 行业前景评分表 |
   | 列表替代 | 内容超过 30 字或需要详细说明 | 风险因素详解 |

4. **避免的表格问题**：
   - ❌ 单元格内换行（使用 `<br>` 在某些渲染器中不兼容）
   - ❌ 过长的单元格内容
   - ❌ 嵌套表格
   - ✅ 复杂内容使用无序列表或子标题代替

5. **推荐表格模板**：

   **基本信息表**：
   ```markdown
   | 项目 | 内容 |
   | :--- | :--- |
   | 公司名称 | xxx |
   | 股票代码 | xxx |
   ```

   **评分表**：
   ```markdown
   | 评分因素 | 评分依据 | 加扣分 | 最终评分 |
   | :--- | :--- | :--- | :--- |
   | 行业前景 | xxx | +0.5 | 4.5 |
   ```

**4.3 文件保存规则**

保存路径：`$OBSIDIAN_VAULT/投资手册/投资报告/<股票名称>(<股票代码>)-投资分析报告.md`

示例：`$OBSIDIAN_VAULT/投资手册/投资报告/海科新源(301292)-投资分析报告.md`

文件处理规则：
- 文件不存在时创建新文件
- 文件已存在时全量更新内容

**5.3 评分与综合评级的关系**

> **⚠️ 关键原则**：所有评分和评级定义均参照 `投资手册/投资评级定义.md` 文件执行。
> 本节仅提供摘要，详细标准请以评级定义文件为准。

**5.3.1 评级等级概览**

| 评级 | 综合评分范围 | 预期收益特征 |
|------|-------------|-------------|
| 强力买入 | 4.5 - 5.0 分 | 显著跑赢大盘，超基准 15%-20%+ |
| 买入 | 4.0 - 4.4 分 | 涨幅高于基准 10%-20% |
| 增持 | 3.5 - 3.9 分 | 小幅跑赢或优于大盘 5%-15% |
| 观望 | 2.5 - 3.4 分 | 与大盘基本持平（±5% 以内） |
| 回避 | 1.0 - 2.4 分 | 跑输大盘或下跌超 5%-15% |

> 详细定义请参考：`[[投资评级定义#评级等级与定义]]`

**5.3.2 股票类型识别**

股票类型包括：亏损股、周期股、转型股、高成长股、成长股、成熟股（共6种）

> **详细识别标准和优先级请参考**：[[投资评级定义#股票类型识别]]
>
> 识别优先级：亏损股 → 周期股 → 转型股 → 高成长股/成长股/成熟股

**5.3.3 动态权重体系**

不同股票类型使用不同的权重体系（6种类型：高成长股、成长股、成熟股、周期股、亏损股、转型股）

> **详细权重表和成长系数机制请参考**：
> - [[投资评级定义#动态权重评分体系]]（权重表）
> - [[投资评级定义#成长系数机制]]（成长系数档位表）
> - [[投资评级定义#转型股评分方法]]（转型股特殊处理）

**5.3.4 评分计算流程**

```
1. 收集数据 → 获取公司财务数据、行业信息、市场数据
2. 分维度评分 → 根据量化标准+加扣分因素评分（1-5分）
3. 识别股票类型 → 根据识别标准判断（6种类型）
4. 计算基础综合评分 → 选择对应权重 → 加权计算
5. 判断是否适用成长系数 → 仅高成长股（增速>50%）和转型股（新业务增速>50%）适用
6. 确定成长系数 → 根据增速查表确定系数（1.00-1.15）
7. 计算最终综合评分 → 基础综合评分 × 成长系数（上限5.0分）
8. 确定评级 → 根据评分范围确定最终评级
```

> **详细计算示例请参考**：[[投资评级定义#使用说明]]（含高成长股、成熟股、转型股案例）
> **成长系数档位表请参考**：[[投资评级定义#成长系数机制]]

> **注意**：
> - `评级星级` 必须为 1-5 的整数，不可省略，否则 base 文件的评级图标公式会返回"未评级"
> - YAML 头部必须包含 `tags: - 投资报告` 以便 base 文件筛选

**5.5 YAML 字段填写顺序**

为确保评分准确性，YAML 字段按以下逻辑顺序填写：

1. **读取评级定义文件**：确保评分标准一致
2. **识别股票类型**：参考 [[投资评级定义#股票类型识别]]
3. **分维度评分**：参考 [[投资评级定义#五维度评分标准]]
4. **计算综合评分**：
   - 权重选择：参考 [[投资评级定义#动态权重评分体系]]
   - 成长系数判断：参考 [[投资评级定义#成长系数机制]]
     - 高成长股（增速>50%）→ 根据增速确定成长系数（1.00-1.15）
     - 转型股（新业务增速>50%）→ 根据新业务增速和占比确定成长系数（1.00-1.15）
     - 其他股票类型 → 成长系数=1.00（不调整）
   - 最终评分 = 基础综合评分 × 成长系数（上限5.0分）
5. **确定评级**：参考 [[投资评级定义#评级与五维度评分对应关系]]
6. **填写 YAML 字段**：`评级分数`、`评级星级` 和 `综合评级`

## 常见问题与解决方案

### 问题 1：更新图标显示为 ⚪ 而非 ✅

**原因**：`是否有更新` 字段值为字符串 `"是"` 而非布尔值 `true`

**解决**：将 YAML 中的 `是否有更新: "是"` 改为 `是否有更新: true`

### 问题 2：报告链接列显示为空

**原因**：缺少 `报告链接` 字段

**解决**：添加 `报告链接: "[[股票名称(股票代码)-投资分析报告]]"` 字段

> ⚠️ **注意**：字段名必须是 `是否有更新`（与 base 文件公式一致），禁止使用 `是否更新`

### 问题 3：缺少五维度评分或评级分数

**原因**：报告未按要求包含完整的五维度评分

**解决**：确保报告"投资建议-综合评级"章节包含：
- 行业前景评分（1-5分）
- 公司竞争力评分（1-5分）
- 成长性评分（1-5分）
- 财务健康度评分（1-5分）
- 估值合理性评分（1-5分）
- 评级分数（五维度平均值）

### 问题 4：评级星级与评级分数不匹配

**原因**：`评级星级` 不是根据 `评级分数` 范围确定，或未使用正确的权重体系

**解决**：
1. 先识别股票类型（参考评级定义文件）
2. 使用对应权重计算加权综合评分
3. 根据评分范围对照表确定评级星级
4. 评级由综合评分决定，必须严格遵循评级定义文件

### 问题 5：评级图标显示"未评级"

**原因**：缺少 `评级星级` 字段或值不在 1-5 范围内

**解决**：
1. 确保 YAML 中包含 `评级星级` 字段
2. 确认 `评级星级` 值为 1-5 的整数
