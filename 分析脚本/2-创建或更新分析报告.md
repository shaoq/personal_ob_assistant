# 股票投资分析报告生成流程

### 步骤 1：确定 Obsidian Vault 路径

在开始分析前，首先确定用户的 Obsidian Vault 实际路径：

**自动检测方式（按优先级尝试）：**

1. 检查常见路径位置：
   - macOS: `~/Documents/Obsidian Vault/` 或 `~/Library/Mobile Documents/iCloud~md~obsidian/Documents/`
   - Windows: `C:\Users\<用户名>\Documents\Obsidian Vault\`
   - Linux: `~/Documents/Obsidian Vault/`

2. 如果自动检测失败，主动询问用户：
   - "请提供您的 Obsidian Vault 的完整路径"

3. 确认路径有效性后，后续操作均使用此路径变量，路径变量标记为：`$OBSIDIAN_VAULT`

---

### 步骤 2：等待用户输入

等待用户提供待分析的股票列表，格式为：`股票代码(股票名称)`

示例输入：
- 单只股票：`301292(海科新源)`
- 多只股票：`301292(海科新源)，600519(贵州茅台)，000858(五粮液)`

---

### 步骤 3：逐只股票执行分析

对每只股票依次执行以下操作,务必逐只执行不要并行：

**3.1 调用分析 Skill**

执行命令：`equity-research:thesis <股票代码>(<股票名称>)`

示例：`equity-research:thesis 301292(海科新源)`

数据要求：必须全面查询所有可用数据，必须补全所有缺失数据字段，必须进行综合分析。

**3.2 报告内容要求**

除了 skill 自带内容外，必须包含但不限以下章节：

- **投资建议-综合评级**：包括“| 行业前景     |
| -------- |
| 公司竞争力    |
| 成长性      |
| 财务健康度    |
| 估值合理性    |”五个维度打分和总评分，并给出"投资评级定义"文件内的对应评级及该股评级理由
- **投资逻辑**：核心投资逻辑的 3-5 点总结
- **投资论点总结**：看多/看空的关键论点归纳
以满足base文件的要求

**3.3 文件保存规则**

保存路径：`$OBSIDIAN_VAULT/投资手册/投资报告/<股票名称>(<股票代码>)-投资分析报告.md`

示例：`$OBSIDIAN_VAULT/投资手册/投资报告/海科新源(301292)-投资分析报告.md`

文件处理规则：
- 文件不存在时创建新文件
- 文件已存在时全量更新内容，并将 YAML 头部的 `是否有更新` 字段设置为 `是`

---

### 步骤 4：YAML 结构适配（重要）

前置检查：
1. 读取 `$OBSIDIAN_VAULT/投资手册/投资分析报告汇总.base` 文件
2. 如果 base 文件存在，提取其定义的字段结构
3. 确保生成的报告 YAML 头部包含所有必需字段

**必需字段清单（必须全部包含）：**

| 字段名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| 股票代码 | string | 6位数字代码 | "300353" |
| 股票名称 | string | 股票中文名称 | "东土科技" |
| 综合评级 | string | 评级定义文件中的评级 | "增持" |
| **评级星级** | **number** | **1-5的整数，base文件评级图标公式依赖此字段** | **4** |
| 评级分数 | number | 五维度平均分（1-5，保留1位小数） | 3.6 |
| 所属行业 | string | 主营业务所属行业 | "工业互联网" |
| 市值 | string | 总市值（带单位） | "153亿元" |
| 报告日期 | string | 分析报告日期 | "2026-03-04" |
| 核心逻辑 | string | 核心投资逻辑一句话总结 | "工业互联网底层根技术龙头..." |
| 操作建议 | string | 操作建议一句话 | "中长线布局，关注业绩兑现" |
| 是否有更新 | string | 新建或更新时设为"是" | "是" |

**⚠️ 关键注意事项：**

1. **`评级星级` 字段必须为1-5的整数**，不可省略，否则 base 文件的评级图标公式会返回"未评级"
2. `评级星级` 与 `评级分数` 的对应关系：

| 评级分数范围 | 评级星级 | 综合评级 |
|--------------|----------|----------|
| 4.5 - 5.0 | 5 | 强力买入 |
| 4.0 - 4.4 | 4 | 买入 |
| 3.5 - 3.9 | 4 | 增持 |
| 2.5 - 3.4 | 3 | 观望 |
| 1.0 - 2.4 | 1-2 | 回避 |

3. YAML 头部必须包含 `tags: - 投资报告` 以便 base 文件筛选

**YAML 模板示例：**
```yaml
---
股票代码: "300353"
股票名称: "东土科技"
综合评级: "增持"
评级星级: 4
评级分数: 3.6
所属行业: "工业互联网"
市值: "153亿元"
报告日期: "2026-03-04"
核心逻辑: "工业互联网底层根技术龙头，鸿道OS+TSN芯片全栈自主可控，政策红利驱动国产替代"
操作建议: "中长线布局，关注2025年业绩兑现及AI机器人OS商业化进展"
是否有更新: "是"
tags:
  - 投资报告
---
```

---

### 步骤 5：清理上下文（循环控制）

每完成一只股票分析后：
1. 保存报告文件
2. 确认文件写入成功
3. 执行 `/clear` 命令清理 Claude 上下文
4. 继续下一只股票的分析

---

### 执行约束

- **路径变量**：使用 `$OBSIDIAN_VAULT` 变量替代硬编码路径，根据实际环境自动匹配
- **专用 Skill**：必须使用 `equity-research:thesis`，不可替换为其他 skill
- **数据完整性**：所有数据字段必须完整，不可遗漏
- **YAML 兼容性**：必须符合 base 文件的字段定义（若存在）
- **上下文管理**：每只股票分析完成后必须执行 `/clear`
- **处理顺序**：按用户输入顺序依次处理，请务必不要开启并行处理
- **完成后校验**：完成分析文档后，请务必检查对应文档成功已创建或更新，若出现问题请检查并修复
- **后续动作**：分析文档完成并检验完成后，务必不要直接执行“投资论点跟踪”等分析等动作，而是等待用户后续输入
