## 📌 流程概述

本流程将股票分析拆分为**3个连续阶段**，**默认连续自动执行完毕**。断点续传机制仅在执行意外中断时启用。

> **⚠️ 自动执行原则**：每个阶段完成后，**必须立即读取并执行下一阶段文档**，无需等待用户确认。

| 阶段 | 包含步骤 | Context消耗 | 输出 | 完成后动作 |
|------|---------|-------------|------|-----------|
| **阶段1** | 数据准备与类型识别 | ~15k-20k tokens | phase1_state.json | **自动读取并执行阶段2文档** |
| **阶段2** | 报告生成 | ~20k-30k tokens | phase2_state.json | **自动读取并执行阶段3文档** |
| **阶段3** | 验证与保存 | ~10k-15k tokens | 最终报告文件 | 流程结束，汇报结果 |

## ⚠️ 关键约束速查表

> **重要**：以下约束在执行过程中必须严格遵守，违反将导致错误结果

| 约束项 | 要求 | 违反后果 | 适用阶段 |
|--------|------|---------|---------|
| **YAML字段顺序** | 必须严格按模板序号排列 | Base视图解析异常 | 阶段3 |
| **YAML字段数量** | 正好12个必需字段+tags | 字段遗漏或多余 | 阶段3 |
| **禁止额外字段** | 仅允许定义的14个字段 | Base视图解析异常 | 阶段3 |
| **字段名正确** | 使用 `评级分数`（非 `评分`） | 评分显示异常 | 阶段3 |
| P2.5数据完整性 | 类型识别必需数据不可缺失>30% | 类型识别失败 | 阶段1 |
| 输入格式 | 每次只输入一只股票 | 处理混乱 | 阶段1 |
| 数据完整性 | P1-P3+P2.5必需数据不可缺失>30% | 评分不准确 | 阶段1 |
| YAML格式 | 严格遵循字段类型定义 | base视图显示异常 | 阶段3 |
| 专用Skill | 必须使用 `equity-research:thesis` | 报告格式不符合要求 | 阶段2 |
| 文件保存 | 必须保存到 `$INVESTMENT_HANDBOOK/投资报告/` | 报告丢失 | 阶段3 |
| 字段名一致性 | 使用 `是否更新`（非 `是否有更新`） | Base视图更新图标显示异常 | 阶段3 |
| 行业阶段字段 | 高成长赛道必须填写起步期/成长期/成熟期 | Base视图无法显示行业阶段 | 阶段3 |
| Markdown格式 | 报告正文必须符合Markdown规范 | 报告渲染异常 | 阶段3 |
| 数据来源 | 优先 `mcp__web-search-prime__webSearchPrime` | 查询失败 | 阶段1 |

## 🔄 断点续传机制

> **执行模式**：流程默认连续执行阶段1→阶段2→阶段3，无需用户干预。
> 断点续传仅在以下情况启用：
> - 执行过程中意外中断（网络问题、手动中止等）
> - 用户主动要求分段执行

### 状态文件存储位置

```
$INVESTMENT_HANDBOOK/.analysis_states/
├── 688809/                      # 股票代码
│   ├── phase1_state.json        # 阶段1状态
│   ├── phase2_state.json        # 阶段2状态
│   └── phase3_state.json        # 阶段3状态
└── 300438/
    └── phase1_state.json        # 未完成的分析
```

### 恢复逻辑

当用户输入股票代码时，系统会自动检测是否存在未完成的状态文件：

| 检测条件 | 提示信息 | 建议操作 |
|---------|---------|---------|
| 存在 phase1 且 status=completed，但无 phase2 | "检测到阶段1已完成，是否从阶段2继续？" | 输入"继续" |
| 存在 phase2 且 status=completed，但无 phase3 | "检测到阶段2已完成，是否从阶段3继续？" | 输入"继续" |
| 所有阶段都已完成 | "该股票分析已完成，是否重新分析？" | 输入"重新分析" |
| 无状态文件 | 开始新的分析流程 | - |

### 状态文件说明

**phase1_state.json** 包含：
- 股票基本信息
- 行业信息（属性、阶段）
- 财务数据（P1-P3）
- 类型识别结果
- 适用的权重体系

**phase2_state.json** 包含：
- 五维度评分详情
- 综合评分计算过程
- 评级结果
- 报告内容

**phase3_state.json** 包含：
- 文件保存信息
- 验证结果
- 异常处理记录

## 🆘 常见问题与解决方案

### 问题 1：阶段执行中断怎么办？

**解决**：利用断点续传机制
1. 重新输入股票代码
2. 系统会检测已完成阶段
3. 输入"继续"从中断处继续

### 问题 2：更新图标显示为 ⚪ 而非 ✅

**原因**：`是否更新` 字段值为字符串而非布尔值

**解决**：将 `是否更新: "是"` 改为 `是否更新: true`

### 问题 3：报告链接列显示为空

**原因**：缺少 `报告链接` 字段

**解决**：添加 `报告链接: "[[股票名称(股票代码)-投资分析报告]]"`

### 问题 4：评级星级与评级分数不匹配

**原因**：评分计算或权重使用错误

**解决**：
1. 重新检查股票类型
2. 使用对应权重体系
3. 确认评级星级与评分范围对应

### 问题 5：行业阶段字段未显示

**原因**：高成长赛道未填写 `行业阶段` 字段

**解决**：在 YAML 中添加 `行业阶段: "成长期"` 等值
