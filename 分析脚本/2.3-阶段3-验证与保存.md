> **重要**：根据验证失败的具体情况，选择对应的异常处理流程。

**7.1 数据缺失过多**

**处理方式**：
1. 中止当前股票分析
2. 向用户报告："必需数据缺失过多（XX%），建议补充以下数据后重新分析："

4. 等待用户确认是否继续或中止

**处理方式**：
```
识别优先级（从高到低）：
1. 亏损股（净利润<0）
2. 周期股（利润波动大）
3. 转型股（新业务占比高）
4. 高成长股（增速>50%）
5. 成长股（增速>15%）
6. 成熟股（其他）
```

**常见错误**：
- 评级星级与评级分数不匹配
- 权重使用错误
- 成长系数应用错误

**处理方式**：
1. 重新识别股票类型
2. 重新选择对应权重体系
3. 重新计算综合评分
4. 重新确定评级星级
5. 更新 YAML 字段
6. 重新执行验证

**7.4 文件保存失败**

**处理方式**：
1. 检查路径是否正确
2. 检查文件权限
3. 检查磁盘空间
4. 重试保存操作
5. 如果仍失败，提示用户手动干预

**7.5 YAML格式错误**

**处理方式**：
1. 根据错误类型修复 YAML 字段
2. 重新保存文件
3. 重新执行验证

**输出结果**：

## 🔧 常见问题与解决方案

### 问题 1：更新图标显示为 ⚪ 而非 ✅

**原因**：`是否更新` 字段值为字符串 `"是"` 而非布尔值 `true`

**解决**：将 YAML 中的 `是否更新: "是"` 改为 `是否更新: true`

---

### 问题 2：报告链接列显示为空

**原因**：缺少 `报告链接` 字段

**解决**：添加 `报告链接: "[[股票名称(股票代码)-投资分析报告]]"` 字段

---

### 问题 3：缺少五维度评分或评级分数

**原因**：报告未按要求包含完整的五维度评分

**解决**：确保报告"投资建议-综合评级"章节包含五维度评分表

---

### 问题 4：评级星级与评级分数不匹配

**原因**：
- `评级星级` 不是根据 `评级分数` 范围确定
- 未使用正确的权重体系
- 股票类型识别错误

**解决**：
1. 重新检查评分计算过程
2. 确认评级星级与评分范围对应

---

### 问题 5：评级图标显示"未评级"

**原因**：缺少 `评级星级` 字段或值不在 1-5 范围内

**解决**：
1. 确保 YAML 中包含 `评级星级` 字段
2. 确认 `评级星级` 值为 1-5 的整数
