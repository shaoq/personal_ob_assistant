```yaml

  股票名称:

  股票代码:

  综合评级:
    displayName: 评级
  formula.评分数值:
    displayName: 评分
  formula.评级图标:
    displayName: 图标
  所属行业:
    displayName: 行业
  formula.市值亿元:
    displayName: 市值

    displayName: 更新

    displayName: 操作建议

### 3.2 关键配置说明

| 配置项 | 说明 |
|--------|------|
| `是否投资报告: file.folder == "投资手册/投资报告"` | **公式判断文件夹**，这是唯一可靠的路径过滤方式 |
| `filters: and: - formula.是否投资报告 == true` | **正确过滤语法**，使用 `filters`（复数）和表达式格式 |
| `groupBy.property: 综合评级` | **按评级分组显示**，直观展示各级别股票 |
| `groupBy.direction: DESC` | **降序排列**，确保最高评级（强力买入）排在最前 |
| 评分公式返回 `null` 而非 `0` | **避免评分 0 记录**，无评级的记录公式返回 null |
| `评分数值` 返回数字类型 | **支持小数和数值排序**，如 3.5、4.2 等 |
| `评分` 字段支持小数 | **支持精细评分**，如 3.5、4.2 等，优先于综合评级映射使用 |

## 综合评级的定义与文件创建

综合评级的定义如下：

| 评级 | 分数 | 含义 | 预期收益 |
|------|------|------|----------|
| 强力买入 | 5 | 最高评级 | 预期未来 6–12 个月显著跑赢大盘（通常超基准指数 15%–20%+） |
| 买入 | 4 | 看好 | 预期跑赢大盘（涨幅高于基准 10%–20%） |
| 增持 | 3 | 谨慎推荐 | 预期小幅跑赢或优于大盘（5%–15%） |
| 观望 | 2 | 中性判断 | 预期与大盘基本持平（±5% 以内） |
| 回避 | 1 | 看空 | 预期跑输大盘或下跌（跌幅超 5%–15%） |

根据综合评级定义创建对应的投资评级定义文件，确保投资分析报告的综合评级在上述定义范围内。

### 精细评分说明

除了使用综合评级映射的整数评分，还可以使用 `评分` 字段提供更精细的评分：

| 场景 | 评分示例 | 说明 |
|------|----------|------|
| 强力买入偏买入 | 4.5 | 略低于强力买入标准，但优于买入 |
| 增持偏买入 | 3.5 | 增持偏积极，接近买入级别 |
| 观望偏增持 | 2.5 | 观望偏积极，有向上空间 |
| 回避偏观望 | 1.5 | 风险较高但非完全回避 |

**使用方式**：在报告的 YAML frontmatter 中添加 `评分` 字段：

```yaml
---
股票名称: 东田微
股票代码: 301183
综合评级: 增持
评分: 3.5
---
```

**评分显示逻辑**：
- 有精细评分时：显示原始数值（如 `3.5`、`4.2`）
- 无精细评分时：根据综合评级显示对应整数分数（`5`、`4`、`3`、`2`、`1`）

**评分优先级**：如果同时存在 `评分` 和 `综合评级`，评分数值将优先使用 `评分` 字段的值。

## 关键注意事项

1. **路径过滤必须使用公式**：Obsidian Base 的 `from` 路径过滤不可靠，必须使用公式 `file.folder == "投资手册/投资报告"` 进行判断
2. **过滤语法必须是 `filters`（复数）**：使用 `filter`（单数）会被忽略，正确语法为：
   ```yaml
   filters:
     and:
       - formula.是否投资报告 == true
   ```
3. **公式默认值返回 `null` 而非 `0`**：当 `综合评级` 不匹配任何预设值时，公式应返回 `null` 而非 `0`，避免产生评分 0 的无效记录
4. **评分使用数值格式**：`评分数值` 公式返回数字类型，支持小数和数值排序
5. **YAML 优先**：遇到结构不完整的报告，必须先补全 YAML 再处理
6. **链接可点击**：确保报告链接在 Obsidian 中可正常跳转
7. **增量更新**：已存在的记录检查是否有内容变化，有则更新「是否更新」字段

## 常见问题与解决方案

### 问题：Base 视图中出现评分 0 的无效记录

**原因分析：**
1. `from` 路径过滤失效，导致 Vault 中所有文件都被统计
2. 使用 `filter`（单数）语法，过滤条件被忽略
3. 公式在无匹配时返回 `0` 而非 `null`

**解决方案：**
```yaml
formulas:
  # 1. 添加文件夹判断公式
  是否投资报告: file.folder == "投资手册/投资报告"
  # 2. 公式默认值返回 null，评分优先使用精细评分字段
  评分数值: if(评分, 评分, if(..., null))

views:
  - filters:  # 3. 使用 filters（复数）
      and:
        - formula.是否投资报告 == true  # 4. 使用表达式语法
```

### 问题：如何使用小数评分？

**场景**：分析报告需要更精细的评分表达，例如"增持偏买入"可评 3.5 分。

**解决方案**：
在报告的 YAML frontmatter 中添加 `评分` 字段：

```yaml
---
股票名称: 东田微
股票代码: 301183
综合评级: 增持
评分: 3.5
---
```

评分数值公式会优先使用 `评分` 字段的值（显示为 `3.5`），若无则回退到综合评级映射（增持 → `3`）。
