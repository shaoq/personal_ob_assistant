## 综合评级的定义与文件创建

本评级体系基于五维度评分系统（行业前景、公司竞争力、成长性、财务健康度、估值合理性），综合评估后给出投资建议。

### 评级等级与定义

| 评级       | 预期收益   | 基准比较                | 风险程度 | 适合投资者  |
| -------- | ------ | ------------------- | ---- | ------ |
| **强力买入** | 显著跑赢大盘 | 超基准指数 15%-20%+      | 中高风险 | 激进型投资者 |
| **买入**   | 看好     | 涨幅高于基准 10%-20%      | 中等风险 | 积极型投资者 |
| **增持**   | 谨慎推荐   | 小幅跑赢或优于大盘 5%-15%    | 中低风险 | 稳健型投资者 |
| **观望**   | 中性判断   | 与大盘基本持平（±5% 以内）     | 低风险  | 保守型投资者 |
| **回避**   | 看空     | 跑输大盘或下跌（跌幅超 5%-15%） | 高风险  | 不建议投资  |

### 评级与五维度评分对应关系

| 评级 | 综合评分范围 | 评分特征 |
| :--- | :----------- | :------- |
| **强力买入** | 4.5 - 5.0 分 | 五维度均优秀，行业景气度极高，公司竞争力突出，成长性强劲，财务健康，估值合理 |
| **买入** | 4.0 - 4.4 分 | 多数维度优秀，行业景气度高，公司竞争力强，成长性好，财务状况良好，估值相对合理 |
| **增持** | 3.5 - 3.9 分 | 整体表现良好，有一定亮点，但存在部分短板（如估值偏高或财务一般） |
| **观望** | 2.5 - 3.4 分 | 优缺点并存，行业或公司存在不确定性，建议等待更好入场时机或观望 |
| **回避** | 1.0 - 2.4 分 | 行业景气度差，公司竞争力弱，财务状况不佳，估值过高或存在重大风险 |

### 股票类型识别

在计算综合评分前，首先识别股票类型，以确定适用的权重体系：

| 股票类型 | 识别条件 | 特征描述 |
| :--- | :--- | :--- |
| **高成长股** | 利润增速>50% 或 营收增速>50% | 处于业绩爆发期，高速扩张 |
| **成长股** | 利润增速30%-50% 或 营收增速30%-50% | 成长动能强劲 |
| **成熟股** | 增速<15%（利润和营收均<15%） | 稳定经营，增长缓慢 |
| **周期股** | 近3年利润增速标准差>50% | 盈利随周期波动 |
| **亏损股** | 净利润为负 且 营收增速>20% | 处于投入期，尚未盈利但增长中 |
| **转型股** | 满足以下任意2条：<br>1. 新业务营收占比>5% 且增速>50%<br>2. 新业务占比年度提升>3个百分点<br>3. 新业务所在行业CAGR>15%<br>4. 研发占比>8% 且新方向投入>50%<br>5. 管理层明确宣布转型战略 | 从传统业务向高成长赛道转型 |

> **识别优先级**：先判断是否为亏损股 → 再判断是否为周期股 → **再判断是否为转型股** → 最后根据增速判断成长/成熟类型
>
> **转型股说明**：转型股识别优先级高于成长类型判断，但低于亏损股和周期股。典型案例如汽车制造企业切入机器人赛道、传统家电企业布局AI等。

### 五维度评分标准

#### 1. 行业前景（1-5分）

**量化评分标准**：
| 分数 | 市场规模CAGR | 政策支持度 | 技术变革 | 竞争格局 |
| :--- | :----------- | :--------- | :------- | :------- |
| 5分 | >20% | 强力支持（国家级战略） | 颠覆性技术突破 | 蓝海市场 |
| 4分 | 10%-20% | 明确支持（产业政策） | 技术快速迭代 | 格局初定 |
| 3分 | 5%-10% | 中性或一般支持 | 技术稳定升级 | 格局稳定 |
| 2分 | 0%-5% | 政策收紧或无支持 | 技术停滞 | 竞争加剧 |
| 1分 | 负增长 | 政策限制/淘汰 | 技术被替代 | 恶性竞争 |

**扣分因素**：
- 行业处于周期高点，产能过剩风险：-0.5分
- 存在重大技术替代风险：-1.0分
- 政策存在不确定性（如集采、补贴退坡）：-0.5分

**加分因素**：
- 行业渗透率<10%，处于早期爆发阶段：+0.5分
- 全球竞争力强，出海空间大：+0.5分

#### 2. 公司竞争力（1-5分）

**量化评分标准**：
| 分数 | 市场地位 | 技术壁垒 | 客户资源 | 毛利率 |
| :--- | :------- | :------- | :------- | :----- |
| 5分 | 全球/国内绝对龙头（份额>30%） | 极高（专利壁垒/独家技术） | 顶级客户 | >50% |
| 4分 | 国内领先（份额15%-30%） | 高（核心专利/技术积累） | 优质客户 | 35%-50% |
| 3分 | 行业中游（份额5%-15%） | 中等（有一定技术积累） | 客户稳定但分散 | 25%-35% |
| 2分 | 行业下游（份额<5%） | 较低（技术门槛不高） | 客户质量一般 | 15%-25% |
| 1分 | 无明显地位 | 无壁垒 | 客户不稳定 | <15% |

**扣分因素**：
- 核心技术依赖外部授权/进口：-0.5分
- 主要客户流失风险（>20%收入来源不稳）：-1.0分
- 产品被替代风险高：-0.5分

**加分因素**：
- 一体化程度高（上下游延伸）：+0.5分
- 研发投入占比>10%：+0.5分
- 拥有独家/垄断性资源：+1.0分
- 海外收入占比>30%（全球化布局）：+0.5分

#### 3. 成长性（1-5分）

**量化评分标准**：
| 分数 | 营收增速 | 利润增速 | 订单/在手订单 | 行业增速对比 |
| :--- | :------- | :------- | :------------ | :----------- |
| 5分 | >50% | >50% | 爆发式增长（同比>100%） | 显著超越行业 |
| 4分 | 30%-50% | 30%-50% | 高速增长（同比50%-100%） | 领先行业 |
| 3分 | 10%-30% | 10%-30% | 稳定增长（同比20%-50%） | 与行业持平 |
| 2分 | 0%-10% | 0%-10% | 增长疲软或下滑 | 落后行业 |
| 1分 | 负增长 | 负增长 | 萎缩 | 显著落后行业 |

**扣分因素**：
- 增速连续2个季度下滑（趋势恶化）：-0.5分
- 增长依赖并购/非经常性损益：-0.5分
- 增长不增收（增收不增利）：-0.5分

**加分因素**：
- 在手订单覆盖年收入>1.5倍：+0.5分
- 新业务/新产品占比快速提升：+0.5分
- 连续3年增速>30%（业绩确定性高）：+0.5分

> **注意**：利润增速优先于营收增速，若利润增速显著高于营收增速（规模效应），可适当加分

#### 4. 财务健康度（1-5分）

**基础评分标准**：
| 分数 | 资产负债率 | 现金流 | 应收账款 | ROE | 净利率 |
| :--- | :--------- | :----- | :------- | :-- | :----- |
| 5分 | <30% | 充裕（经营现金流/净利润>1.2） | 健康（周转天数<60天） | >20% | >20% |
| 4分 | 30%-50% | 良好（经营现金流/净利润>1.0） | 可控（周转天数60-90天） | 15%-20% | 15%-20% |
| 3分 | 50%-60% | 一般（经营现金流/净利润>0.8） | 偏高（周转天数90-120天） | 10%-15% | 10%-15% |
| 2分 | 60%-70% | 紧张（经营现金流/净利润<0.8） | 高企（周转天数120-180天） | 5%-10% | 5%-10% |
| 1分 | >70% | 为负或持续恶化 | 严重（周转天数>180天） | <5% | <5% |

**扣分因素**（经营风险评估）：
- 客户集中度 >50%（前五大客户占比）：-0.5分
- 客户集中度 >70%：-1.0分
- 客户集中度 >80%：-1.5分
- 供应商集中度 >60%：-0.5分
- 存货周转率显著下降（同比>20%）：-0.5分
- 存在重大诉讼或监管处罚：-1.0分

**加分因素**：
- 现金储备充足（现金/总资产>30%）：+0.5分
- 连续3年分红且股息率>2%：+0.5分
- 无有息负债：+0.5分

> **注意**：最终评分 = 基础评分 + 加分 - 扣分，最低不低于1分，最高不超过5分

#### 5. 估值合理性（1-5分）

**估值评估原则**：
- 优先使用**动态PE**（基于未来12个月预期EPS计算），更能反映成长股的真实估值
- 对于高成长公司（增速>30%），重点参考**PEG指标**
- 对于成熟公司（增速<15%），重点参考**PE/PB相对行业均值的位置**

**基础评分标准**（优先使用动态估值）：
| 分数 | 动态PE相对行业 | PEG | PB相对行业 | 说明 |
| :--- | :------------ | :-- | :--------- | :--- |
| 5分 | 低于行业均值20%以上 | <1.0 | 低于行业均值 | 安全边际充足，显著低估 |
| 4分 | 接近或略低于行业均值（±10%） | 1.0-1.5 | 接近行业均值 | 估值合理 |
| 3分 | 高于行业均值10%-30% | 1.5-2.0 | 高于行业均值10%-30% | 估值偏高但可接受（需成长性支撑） |
| 2分 | 高于行业均值30%-50% | 2.0-3.0 | 高于行业均值30%-50% | 估值泡沫明显 |
| 1分 | 高于行业均值50%以上 | >3.0 | 高于行业均值50%以上 | 严重高估 |

**调整因素**：
- 业绩增速>50%，可适当放宽PE要求：+0.5分（但PEG仍需<2）
- 处于行业爆发期，市场给予估值溢价：+0.5分（最高不超过4分）
- 近期有重大利空导致股价暴跌：-0.5分（可能存在价值陷阱）
- 股价显著高于相对估值区间（DCF/NAV等）：-0.5分

**静态PE与动态PE的使用场景**：
| 场景 | 使用指标 | 原因 |
| :--- | :------- | :--- |
| 成长股（增速>30%） | 动态PE + PEG | 业绩快速增长可消化高估值 |
| 周期股 | 静态PE（取周期均值） | 盈利波动大，单年PE失真 |
| 成熟股（增速<15%） | 静态PE + PB | 增长稳定，历史数据更可靠 |
| 亏损或微利股 | PB + PS | PE失效，用其他指标替代 |

> **注意**：对于高成长公司，即使静态PE较高（如100倍以上），若动态PE合理（如30倍以下）且PEG<1.5，仍可给予3-4分

### 动态权重评分体系

#### 权重分配原则

不同类型股票的核心投资逻辑不同，因此各维度权重应动态调整：

| 股票类型 | 行业前景 | 公司竞争力 | 成长性 | 财务健康度 | 估值合理性 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **高成长股**（增速>50%） | 25% | 20% | 35% | 10% | 10% |
| **成长股**（增速30%-50%） | 22% | 22% | 30% | 13% | 13% |
| **成熟股**（增速<15%） | 18% | 22% | 10% | 28% | 22% |
| **周期股** | 25% | 20% | 12% | 23% | 20% |
| **亏损股**（营收增长中） | 28% | 25% | 30% | 10% | 7% |
| **转型股** | 28% | 22% | 25% | 15% | 10% |

#### 权重设计逻辑

##### 高成长股（增速>50%）
- **成长性35%**：核心优势维度，给予最高权重
- **行业前景25%**：成长股高度依赖行业景气度
- **公司竞争力20%**：执行力和技术壁垒重要
- **财务健康度10%**：扩张期财务指标天然偏低，降低权重
- **估值合理性10%**：静态PE失真，降低权重

##### 成熟股（增速<15%）
- **财务健康度28%**：成熟公司财务稳健性是核心
- **估值合理性22%**：静态估值指标更可靠
- **成长性10%**：增长已不是核心驱动力

##### 周期股
- **行业前景25%**：周期位置决定盈利
- **财务健康度23%**：穿越周期的能力关键

##### 亏损股（营收增长中）
- **行业前景28%**：亏损股高度依赖行业赛道和景气度
- **公司竞争力25%**：技术和市场地位决定能否盈利
- **成长性30%**：营收增速是核心观察指标
- **财务健康度10%**：现金流和现金储备比利润更重要
- **估值合理性7%**：PE失效，主要参考PS/PB，权重最低

#### 成长系数调整（补偿上限机制）

对于**高成长股**，应用成长系数作为**补偿机制**，而非额外加成：

```
成长系数 = 1 + (成长性得分 - 3) × 0.05

最终评分 = min(加权综合评分 × 成长系数, 等权重平均 + 0.3)
```

**设计逻辑**：
- 高成长股的财务健康度和估值合理性往往偏低（扩张期负债高、静态PE失真）
- 动态权重已将成长性权重提升至35%，是对成长优势的主要体现
- 成长系数用于**补偿**财务/估值的拖累，而非**额外加成**
- 最终评分不超过"等权重平均 + 0.3"，避免三重加成导致评分过高

| 成长性得分 | 成长系数 | 补偿效果 |
| :--- | :--- | :--- |
| 5分 | 1.10 | 可补偿财务/估值拖累，最高拉回0.3分 |
| 4分 | 1.05 | 适度补偿 |
| 3分 | 1.00 | 无补偿 |
| 2分 | 0.95 | 成长疲软，适度扣减 |
| 1分 | 0.90 | 负增长，显著扣减 |

> **注意**：应用成长系数后，最终评分仍需限制在1-5分范围内。

#### 转型股评分方法

转型股是从传统业务向高成长赛道转型的企业，需要特殊的评分方法来准确反映其投资价值。

##### 行业前景"双轨制"评分

转型股需要同时对**原主业**和**新业务**分别评分，然后加权：

```
行业前景综合评分 = 原主业评分 × 原主业权重 + 新业务评分 × 新业务权重
```

**权重确定原则**：

| 转型阶段 | 原主业权重 | 新业务权重 | 判断依据 |
| :--- | :--- | :--- | :--- |
| 早期（新业务占比<10%） | 70% | 30% | 新业务贡献小但潜力大 |
| 中期（新业务占比10%-30%） | 50% | 50% | 双轮驱动 |
| 后期（新业务占比>30%） | 30% | 70% | 新业务已成为核心 |

**示例**：
- 汽车制造企业切入机器人赛道
- 原主业（汽车）：3分（成熟行业）
- 新业务（机器人）：5分（高景气赛道）
- 新业务占比15%（中期转型）
- 行业前景综合 = 3×50% + 5×50% = 4分

##### 成长性评分（加入前瞻性指标）

转型股的成长性评分应结合**当前增速**和**未来潜力**：

| 分数 | 当前增速 | 前瞻指标（满足任一） |
| :--- | :--- | :--- |
| 5分 | >30% 或 新业务增速>100% | 新业务订单爆发（>100%）、产能已就绪 |
| 4分 | 15%-30% 或 新业务增速>50% | 新业务订单高增长、客户拓展顺利 |
| 3分 | 5%-15% 或 新业务增速>30% | 新业务稳步推进、有明确里程碑 |
| 2分 | 0%-5% | 新业务进展缓慢 |
| 1分 | 负增长 | 转型失败或停滞 |

**加分因素**（转型股专属）：

| 加分因素 | 加分 |
| :--- | :--- |
| 新业务在手订单覆盖预期产能>80% | +0.5分 |
| 新业务已获得头部客户验证 | +0.5分 |
| 新业务毛利率高于原主业 | +0.5分 |
| 管理层有成功转型历史 | +0.5分 |

**扣分因素**（转型股专属）：

| 扣分因素 | 扣分 |
| :--- | :--- |
| 转型已持续3年以上但新业务仍未盈利 | -0.5分 |
| 新业务资本开支过大导致现金流紧张 | -1.0分 |
| 原主业快速恶化拖累转型资源 | -0.5分 |

##### 转型系数调整（补偿上限机制）

转型股应用转型系数作为**补偿机制**，而非额外加成：

```
转型系数 = 1 + (新业务评分 - 3) × 0.03 × (新业务权重/50%)

最终评分 = min(加权综合评分 × 转型系数, 等权重平均 + 0.3)
```

**设计逻辑**：
- 转型股的财务健康度和估值合理性往往偏低（转型投入大、估值波动）
- 转型系数用于**补偿**财务/估值的拖累，而非**额外加成**
- 最终评分不超过"等权重平均 + 0.3"，避免过度加成

| 新业务前景评分 | 新业务占比 | 转型系数 | 补偿效果 |
| :--- | :--- | :--- | :--- |
| 5分 | >30% | 1.12 | 转型成功在望，可补偿财务/估值拖累 |
| 5分 | 10%-30% | 1.06 | 转型中期，适度补偿 |
| 4分 | 10%-30% | 1.03 | 稳健转型中 |
| 3分 | 任意 | 1.00 | 无补偿 |
| 2分 | 任意 | 0.97 | 转型受阻，适度扣减 |
| 1分 | 任意 | 0.94 | 转型失败风险高，显著扣减 |

##### 转型股权重设计逻辑

| 维度 | 权重 | 设计逻辑 |
| :--- | :--- | :--- |
| 行业前景 | 28% | 双赛道前景，权重提高 |
| 公司竞争力 | 22% | 执行力是转型成功关键 |
| 成长性 | 25% | 兼顾当前增速和未来潜力 |
| 财务健康度 | 15% | 转型期财务压力天然偏大，适度降低 |
| 估值合理性 | 10% | 转型期估值波动大，降低权重 |

> **注意**：应用转型系数后，最终评分仍需限制在1-5分范围内。

#### 亏损股估值方法

对于尚未盈利的公司，估值评估需要特殊处理：

| 方法 | 说明 | 适用场景 |
| :--- | :--- | :--- |
| **PS（市销率）** | 市值/营收，替代PE | 营收稳定增长但未盈利 |
| **PB（市净率）** | 市值/净资产 | 资产较重的公司 |
| **现金消耗率** | 月均烧钱速度 | 评估生存能力 |
| **现金跑道** | 账面现金可支撑月数 | 评估融资紧迫性 |

### 使用说明

#### 评分计算流程

1. **收集数据**：获取公司财务数据、行业信息、市场数据
2. **分维度评分**：
   - 根据量化标准确定基础评分
   - 应用加分/扣分因素
   - 确保最终评分在1-5分范围内
3. **识别股票类型**：根据增速指标判断股票类型（高成长股/成长股/成熟股/周期股/亏损股/转型股）
4. **计算综合评分**：
   - 根据股票类型选择对应权重
   - 计算加权综合评分
   - 对高成长股应用成长系数调整
   - 对转型股应用转型系数调整
5. **确定评级**：根据综合评分范围确定最终评级

#### 重要提醒

1. **一致性原则**：同一公司在不同时期评分时，应使用相同的评分标准
2. **数据时效性**：使用最新的财务数据和市场信息
3. **主观判断最小化**：严格按照量化标准评分，减少主观因素
4. **特殊情况说明**：如有偏离标准的评分，需在报告中说明原因
5. **定期回顾**：建议每季度或重大事件后重新评估评级
根据综合评级定义创建对应的投资评级定义文件，确保投资分析报告的综合评级在上述定义范围内。
### 精细评分使用方式（Base 文件特有）

除了使用综合评级映射的整数评分，还可以使用 `评分` 字段提供更精细的评分（如 3.5、4.2）。在报告的 YAML frontmatter 中添加 `评分` 字段：

```yaml
---
股票名称: 东田微
股票代码: 301183
综合评级: 增持
评分: 3.5
---
```

**评分显示逻辑**：
- 有精细评分时：显示原始数值（如 `3.5`、`4.2`）
- 无精细评分时：根据综合评级显示对应整数分数（`5`、`4`、`3`、`2`、`1`）

**评分优先级**：如果同时存在 `评分` 和 `综合评级`，评分数值将优先使用 `评分` 字段的值。

## 关键注意事项

1. **路径过滤必须使用公式**：Obsidian Base 的 `from` 路径过滤不可靠，必须使用公式 `file.folder == "投资手册/投资报告"` 进行判断
2. **过滤语法必须是 `filters`（复数）**：使用 `filter`（单数）会被忽略，正确语法为：
   ```yaml
   filters:
     and:
       - formula.是否投资报告 == true
   ```
3. **公式默认值返回 `null` 而非 `0`**：当 `综合评级` 不匹配任何预设值时，公式应返回 `null` 而非 `0`，避免产生评分 0 的无效记录
4. **评分使用数值格式**：`评分数值` 公式返回数字类型，支持小数和数值排序
5. **YAML 优先**：遇到结构不完整的报告，必须先补全 YAML 再处理
6. **链接可点击**：确保报告链接在 Obsidian 中可正常跳转
7. **增量更新**：已存在的记录检查是否有内容变化，有则更新「是否更新」字段

## 常见问题与解决方案

### 问题：Base 视图中出现评分 0 的无效记录

**原因分析：**
1. `from` 路径过滤失效，导致 Vault 中所有文件都被统计
2. 使用 `filter`（单数）语法，过滤条件被忽略
3. 公式在无匹配时返回 `0` 而非 `null`

**解决方案：**
```yaml
formulas:
  # 1. 添加文件夹判断公式
  是否投资报告: file.folder == "投资手册/投资报告"
  # 2. 公式默认值返回 null，评分优先使用精细评分字段
  评分数值: if(评分, 评分, if(..., null))

views:
  - filters:  # 3. 使用 filters（复数）
      and:
        - formula.是否投资报告 == true  # 4. 使用表达式语法
```

### 问题：如何使用小数评分？

**场景**：分析报告需要更精细的评分表达，例如"增持偏买入"可评 3.5 分。

**解决方案**：
在报告的 YAML frontmatter 中添加 `评分` 字段：

```yaml
---
股票名称: 东田微
股票代码: 301183
综合评级: 增持
评分: 3.5
---
```

评分数值公式会优先使用 `评分` 字段的值（显示为 `3.5`），若无则回退到综合评级映射（增持 → `3`）。

```
